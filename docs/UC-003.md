# UC-003 统计结果可视化

## 目标

用户可以将 UC-002 生成的 CSV 统计报表转换为自包含的 HTML 柱状图，并由工具自动使用本地浏览器打开。

## 命令示例

```powershell
go run ./cmd/log-tool --visual --csv ./report.csv
```

## 参数

- `--visual`：启用统计结果可视化模式。
- `--csv`：UC-002 生成的 CSV 报表路径，使用 `--visual` 时必填。

`--visual` 只能与 `--csv` 配合使用，不能与 `--file`、`--level`、`--stat`、`--start`、`--end` 或 `--output` 混用。

## 处理流程

1. Go 命令层校验参数组合和 CSV 报表格式。
2. Go 通过 `os/exec` 启动 Python 3 可视化脚本。
3. Python 使用标准库 `csv` 读取报表，并使用 `matplotlib` 生成“日志级别 vs 数量”柱状图。
4. Python 将柱状图以 Base64 PNG 形式嵌入自包含 HTML，不生成需要单独管理的图片文件。
5. HTML 提供放大、缩小、重置和下载 PNG 按钮。
6. HTML 成功生成后，Go 尝试通过当前操作系统的默认浏览器打开。

## CSV 输入约定

- 表头必须为 `level,count,percentage`。
- 每个数据行必须恰好包含三个字段。
- `level` 不能为空。
- `count` 必须是非负整数。
- `percentage` 必须是 `0` 到 `100` 之间的有限数值。
- UC-002 生成的仅含表头的空报表是合法输入，应生成显示“暂无统计数据”的 HTML 页面。
- 无法读取、表头错误或数据字段无效时，统一视为报表无效或损坏。

## HTML 输出约定

- 输出文件与 CSV 位于同一目录，并使用相同文件名主体；例如 `report.csv` 生成 `report.html`。
- HTML 使用 UTF-8 编码，并包含完整标题、图表说明和交互按钮。
- HTML 完全自包含，不依赖远程 JavaScript、CSS、图片或在线服务。
- 有数据时按照 CSV 中的行顺序显示柱状图。
- 没有数据时不绘制空坐标轴，而是显示明确的“暂无统计数据”提示。
- 浏览器无法自动打开时保留已经生成的 HTML，并返回明确诊断。

## 依赖约定

- 运行环境需要 Python 3 和 `matplotlib`。
- 不引入 Plotly、mpld3 或其他 Python 第三方可视化依赖。
- Go 单元测试通过注入子进程执行器和浏览器打开器完成，不要求 Ubuntu CI 预装 `matplotlib`。

## 异常处理

- CSV 文件无效或损坏：提示“报表文件无效或损坏，请重新生成”。
- Python 或 `matplotlib` 缺失：提示“请安装Python及matplotlib库后重试”。
- HTML 无法写入：返回包含目标路径和底层原因的明确错误。
- 浏览器无法打开：HTML 保持可用，提示用户手动打开生成的文件。

## 验收标准

1. `--visual --csv report.csv` 能生成同目录的 `report.html`。
2. Go 使用子进程执行 Python 脚本，Python 使用 `matplotlib` 生成柱状图。
3. 柱状图横轴为日志级别，纵轴为数量。
4. HTML 页面支持放大、缩小、重置和下载图表 PNG。
5. HTML 不依赖网络资源，可以离线打开。
6. 生成成功后工具会尝试使用默认浏览器自动打开 HTML。
7. UC-002 的空报表能够生成“暂无统计数据”页面。
8. 损坏 CSV 和缺失 Python/`matplotlib` 时返回规定的友好提示。
9. 可视化模式参数不能与过滤或统计模式参数混用。
10. UC-001 和 UC-002 的现有行为保持兼容。
11. 核心 Go 模块单元测试覆盖率不低于 85%。
12. README 和用户操作说明包含 UC-003 的依赖、命令和故障排查方式。

## 非目标

- 不提供 Web 服务或长期运行的本地 HTTP 服务。
- 不提供多个图表类型、实时刷新或拖拽式仪表盘。
- 不在本用例中自动安装 Python 或 `matplotlib`。
