# UC-002 日志统计与 CSV 报告

## 目标

用户可以统计指定日期范围内各日志级别的数量和占比，并生成 CSV 报告。

## 命令示例

```powershell
go run ./cmd/log-tool `
  --file ./testdata/sample.log `
  --stat `
  --start 2026-03-01 `
  --end 2026-03-02 `
  --output report.csv
```

## 参数

- `--file`：日志文件路径，必填。
- `--stat`：启用统计模式。
- `--start`：开始日期，格式为 `YYYY-MM-DD`。
- `--end`：结束日期，格式为 `YYYY-MM-DD`。
- `--output`：CSV 输出路径，默认是 `report.csv`。

## 验收标准

1. 开始日期和结束日期均包含在统计范围内。
2. 统计日期范围内的全部日志级别。
3. `--stat` 不能与 `--level` 同时使用。
4. CSV 包含 `level`、`count`、`percentage` 字段。
5. 日期格式错误时返回明确错误。
6. 开始日期晚于结束日期时返回明确错误。
7. 没有符合条件的数据时仍生成空报告，并输出提示信息。
8. 格式错误的日志行沿用 UC-001 行为：输出诊断并继续处理。
9. 测试覆盖率目标不低于 85%。

## 待确认的输出细节

- 空报告是仅保留 CSV 表头，还是生成零字节文件。
- `percentage` 保留多少位小数。
- 大小写不同的日志级别是否合并统计。
